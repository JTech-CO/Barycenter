**버전**: 0.1
**작성일**: 2026년 7월 19일
**상태**: 구현 착수 기준선
**관계 문서**: `AGENTS.md`, `PROGRESS.md`, `Barycenter_하네스.md`, `Barycenter_기술백서.md`, `Barycenter_디자인백서.md`

> 이 문서는 구현 순서와 산출물, 진입 조건, 완료 게이트를 정의한다. 제품·물리 요구사항은 백서, 반복 작업 규율과 중단 기준은 하네스, 현재 상태와 결정 이력은 `PROGRESS.md`가 담당한다.

## 1. 계획 원칙

### 1.1 출시 경로

핵심 v1 경로는 다음과 같다.

```text
M0 → M1 → M2 → M3 → M4 → M5 → M7 → Core v1
                              └→ M6 (선택, Core v1 비차단)
```

- M0은 이번 문서화와 Codex 전환으로 완료한다.
- M3 이전에는 UI 완성도보다 수치 정확성을 우선한다.
- M4의 회전 좌표계·CR3BP·라그랑주점은 제품 핵심 약속이므로 v1 필수다.
- M6 자연어 시나리오는 선택 기능이다. 실패하거나 배포하지 않아도 코어 v1은 완결되어야 한다.
- 가속 경로는 정확성 기준인 CPU 직접합산을 먼저 굳힌 뒤 M7에서 실제 프로파일에 근거해 도입한다.

### 1.2 범위 기준

- 내부 물리 상태는 3D이며 v1 그래픽은 기준면으로 정사영한다.
- 공개 단위는 AU·yr·M☉, 중력상수는 `G = 4π²`다.
- v1 케플러 요소 입력은 타원 궤도 `0 ≤ e < 1`로 제한한다. 포물선·쌍곡선 운동은 직교 상태벡터로 표현할 수 있으나 요소 변환은 후속 범위다.
- 기본 적분기는 고정 스텝 leapfrog, 고정밀 선택지는 Yoshida4다. RK4는 비교·교육·검증용이다.
- 충돌 병합과 GPU 힘 계산은 명시적으로 후순위다. 측정 또는 사용자 결정 없이 코어 v1의 필수 범위로 끌어오지 않는다.

### 1.3 산출물 우선순위

1. 재현 가능한 수치 코어와 자동 검증
2. CR3BP와 라그랑주점 분석 정확성
3. 코어 상태를 왜곡하지 않는 인터랙티브 제품 수직 슬라이스
4. 저장·공유·접근성·브라우저 호환성
5. 측정에 근거한 성능 최적화
6. 선택적 외부 AI 연동

## 3. 전 마일스톤 공통 수치 계약

### 3.1 상태와 직렬화

- `Scenario`의 직렬화 경계는 사람이 읽고 검증할 수 있는 `Body[]`와 버전 필드를 사용한다.
- 실행 시점에는 위치·속도·질량을 `Float64Array` 기반의 조밀한 배열로 변환한다. 직렬화 객체와 런타임 배열을 같은 소유권으로 섞지 않는다.
- 천체 ID와 배열 인덱스의 대응은 한 곳에서 관리하고, 천체 삭제·병합이 없는 v1 코어에서는 적분 중 순서를 고정한다.
- 초기조건, 적분기, `dt`, substeps, softening, 좌표계, 렌더 재현 설정을 모두 공유 데이터에 포함한다.

### 3.2 힘과 에너지의 일치

Plummer softening을 사용할 때 힘과 진단 에너지는 같은 퍼텐셜 쌍을 사용한다.

```text
Uε = -Σ(i<j) G mi mj / sqrt(rij² + ε²)
```

softening이 적용된 힘에 비softened 퍼텐셜을 조합하지 않는다. 이 계약은 M2 단위 테스트와 M3 장기 보존 테스트 양쪽에서 검증한다.

### 3.3 보존 오차 정의

- 에너지는 signed 변화량과 절댓값을 모두 보관한다. `|E0|`가 특성 에너지 대비 너무 작으면 단순 `ΔE/E0`를 표시하지 않고 scale-normalized 절대 오차로 전환한다.
- 각운동량은 `L0 = 0`인 figure-8 계에서도 정의되도록 `||L-L0|| / Σ mi||ri×vi||` 형태의 scale-normalized 오차를 사용한다.
- 질량중심 드리프트는 정지한 `com0`가 아니라 관성계의 예상 운동 `com0 + (P0/M)t`와의 차이로 계산한다.
- `fixed=true`인 천체가 있으면 계는 외력 제약을 받은 것으로 표시한다. 이 경우 폐쇄계 에너지·운동량 보존을 통과 조건으로 사용하지 않는다.

정확한 수치 floor와 허용오차는 M3의 `dt` 수렴 실험으로 결정해 검증 fixture와 `PROGRESS.md`에 함께 고정한다.

### 3.4 결정론과 테스트 계층

- CPU 기준 경로는 고정된 천체 순서, 고정 `dt`, 고정 substeps, 고정 연산 순서를 사용한다.
- 빠른 검증은 단위·짧은 궤도·대표 회귀를 포함해 모든 변경에서 실행한다.
- 장기 검증은 10²–10³ 주기 적분과 전체 리포트를 별도 명령으로 실행한다. 릴리스 전에는 필수이며, CI 비용이 커지면 예약 실행으로 분리한다.
- GPU 경로는 bitwise 재현을 요구하지 않고 CPU 기준과의 수치 parity를 요구한다.

## 5. M1 — 기반·계약·스캐폴딩

### 진입 조건

M0 게이트 통과.

### 목표

기능 개발 전에 설치·빌드·정적 분석·테스트가 재현되고, 코어와 UI의 경계가 자동으로 강제되는 최소 저장소를 만든다.

### 작업 패키지

- **M1-01 앱 기반**: pnpm, Vite, React, JavaScript/ESM, JSDoc `checkJs`, Zustand 구성
- **M1-02 검증 도구**: ESLint, Vitest, 테스트 커버리지 기반과 `lint`·`typecheck`·`test`·`build`·`verify` 스크립트
- **M1-03 디렉터리 경계**: `src/core`, `src/runtime`, `src/scenarios`, `src/state`, `src/render`, `src/components`, `src/utils`
- **M1-04 코어 import 규칙**: `src/core`에서 React, Zustand, DOM, Canvas, 브라우저 전역 의존 차단
- **M1-05 최소 수직 테스트**: 단위 상수와 Vec3의 내적·외적·정규화·영벡터 정책 테스트
- **M1-06 저장소 위생**: lockfile, `.gitignore`, 테스트/벤치 산출물 위치, 비밀정보 제외 규칙

### 완료 게이트

1. 깨끗한 설치 후 `pnpm verify`가 성공한다.
2. `src/core`에서 금지 import를 넣은 검증 fixture가 lint 실패한다.
3. Vec3와 단위 상수 테스트가 통과하고 `G = 4π²`가 단일 정의를 가진다.
4. production build를 로컬에서 열었을 때 오류 없는 빈 해석 프레임이 표시된다.
5. 실제 선택한 Node·pnpm·주요 패키지 버전과 명령을 `PROGRESS.md`에 기록한다.

### 비범위

실제 중력 계산, 완성 UI, 성능 최적화, 외부 AI 연동.

## 6. M2 — 결정론적 수치 코어

### 진입 조건

M1 게이트 통과.

### 목표

브라우저나 React 없이 테스트 가능한 직접합산 N-body 코어를 완성한다.

### 작업 패키지

- **M2-01 상태 계약**: massive/tracer가 명시된 `Body`, 런타임 SoA 상태, `World`, `SimConfig`, 검증 오류 형식
- **M2-02 벡터·단위**: 할당을 통제하는 3D 벡터/TypedArray 유틸과 단위 변환
- **M2-03 힘 계산**: massive 쌍의 대칭 직접합산, tracer의 source 제외/target 가속, softening, 가속도 출력, 유한값 검사
- **M2-04 적분기**: leapfrog KDK, Yoshida4 합성, 비교용 RK4, 공통 integrator 인터페이스
- **M2-05 궤도 요소**: anomaly 종류(`true`/`mean`)가 명시된 타원 케플러 요소 ↔ 직교 상태, 각도 정규화, 평균근점 이각의 고이심률 수렴 보호
- **M2-06 진단**: 총에너지·선운동량·각운동량·질량중심과 scale-aware 오차
- **M2-07 월드 오케스트레이션**: 순수 step, reset, snapshot, 발산 안전정지, 오류 원인 보존

### 완료 게이트

1. 두 천체 쌍에서 `Σ mi ai`가 부동소수 허용범위 안에서 0이고 힘 방향이 해석값과 일치한다.
2. 겹친 위치, 극단 질량비, 고이심률 입력에서 NaN/Inf가 생성되지 않으며 무효 입력은 step 전에 거부된다. 질량 0 tracer는 massive source에 의해 가속되지만 massive 궤도와 보존량을 바꾸지 않는다.
3. 케플러 → 상태 → 케플러 왕복 오차가 각도 wrapping 후 `≤ 1e-9`다.
4. 동일 초기조건을 CPU 고정 경로로 3회 적분했을 때 저장된 상태가 정해진 결정론 허용오차 안에서 같다.
5. leapfrog 시간반전과 `dt` 축소 수렴 테스트가 통과한다.
6. softening 힘과 진단 퍼텐셜의 미분 관계를 수치 차분으로 확인한다.
7. 발산 안전정지는 실패 step을 커밋하지 않고 마지막 유효 스냅샷과 원인을 반환한다.

### 비범위

장기 허용오차 확정, 회전 좌표계, Barnes–Hut/GPU, 렌더링.

## 7. M3 — 물리 정확성·장기 보존 검증

### 진입 조건

M2 게이트 통과.

### 목표

"계산이 실행된다"가 아니라 알려진 해와 보존 법칙을 재현한다는 증거를 자동 생성하고, 이후 회귀 테스트가 사용할 허용오차를 고정한다.

### 기준 시나리오

- 원형 이체계
- 편심 이체계
- 큰 질량비의 행성계
- figure-8 삼체
- 대칭 정지/저운동량 다체계
- softening 근접 조우 안전성 케이스

각 fixture에는 초기조건 출처, 단위 변환, 기준 주기, `dt`, substeps, 기대 보존량을 함께 둔다.

### 작업 패키지

- **M3-01 검증 러너**: 시나리오×적분기×`dt` 행렬과 JSON/Markdown 리포트
- **M3-02 해석해 비교**: 케플러 주기, 반장축, 이심률, 위상 오차
- **M3-03 장기 보존**: signed 에너지 오차, 각운동량 오차, 예상 COM 경로 오차
- **M3-04 수렴 차수**: `dt`, `dt/2`, `dt/4`에서 leapfrog/Yoshida4 오차 기울기 확인
- **M3-05 비교 시연 근거**: 동일 문제의 leapfrog와 RK4 장기 에너지 거동 리포트
- **M3-06 허용오차 고정**: fixture별 기준을 코드와 `PROGRESS.md` 결정 로그에 기록

### 완료 게이트

1. 이체계 수치 주기 상대오차가 `≤ 1e-4`이고 궤도 형상 오차가 고정된 fixture 기준 이내다.
2. leapfrog 이체계를 최소 10³ 주기 적분했을 때 signed 에너지 오차의 선형 추세가 peak-to-peak 진동 포락선의 10%를 넘지 않는다.
3. figure-8을 최소 10² 기준 주기 적분했을 때 궤도 붕괴·천체 유실·NaN이 없고 재귀 근접 오차가 고정 기준 이내다.
4. 각운동량과 예상 COM 경로의 scale-normalized 오차가 모든 폐쇄계 fixture 기준 이내다.
5. `dt` 축소 실험이 반올림 오차 지배 구간 전까지 적분기 차수와 일관된 감소를 보인다.
6. RK4 비교는 교육 근거 리포트로 남기되, 특정 `dt`에서 반드시 단조 드리프트해야 한다는 취약한 CI assert로 만들지 않는다.
7. 빠른 회귀 명령과 장기 검증 명령이 분리되어 있고 둘 다 재현 방법이 문서화된다.

### 출시 표식

이 게이트 통과 시 **Core Alpha**로 표기한다. 이후 UI나 가속 경로는 이 결과를 기준선으로 사용한다.

## 8. M4 — 회전 좌표계·CR3BP·라그랑주점

### 진입 조건

M3 게이트 통과.

### 목표

제품 목적에 명시된 제한 삼체 문제와 라그랑주점 분석을 수치 코어 위에 정확하게 구축한다.

### 작업 패키지

- **M4-01 좌표계**: 관성계·질량중심계·두 주천체 공회전계의 위치/속도 정·역변환
- **M4-02 CR3BP 정규화**: 질량비 `μ`, massless tracer, 길이·시간 스케일과 AU·yr·M☉ 경계 변환
- **M4-03 유효 퍼텐셜**: effective potential과 gradient, zero-velocity contour 데이터
- **M4-04 야코비 상수**: CR3BP 상태의 Jacobi constant와 드리프트 계측
- **M4-05 라그랑주점**: L1–L3 bracketed root solve, L4–L5 해석 위치
- **M4-06 분석 프리셋**: 최소 두 개의 검증 가능한 CR3BP 시나리오

### 완료 게이트

1. 관성계 ↔ 회전계 정·역변환의 위치·속도 왕복 오차가 `≤ 1e-12`다.
2. L1–L5에서 유효 퍼텐셜 gradient norm이 고정 residual 기준 이내다.
3. L4/L5 좌표가 해석 위치와 일치하고, L1–L3 root solver가 bracket과 수렴 실패 원인을 보존한다.
4. 기준 CR3BP 궤도에서 Jacobi constant 오차가 선택한 적분기·`dt` 기준 안에서 유계다.
5. 등고선 샘플은 유한값이며 해상도·범위를 바꿔도 라그랑주점 위치와 좌표계 정의가 변하지 않는다.
6. 모든 분석 모듈은 DOM/Canvas 없이 테스트된다.

### 출시 표식

이 게이트 통과 시 **Analysis Alpha**로 표기한다.

## 9. M5 — 인터랙티브 해석 도구

### 진입 조건

M4 게이트 통과.

### 목표

검증된 코어를 사용해 프리셋 로드부터 편집·재생·계측·공유까지 이어지는 제품 수직 슬라이스를 완성한다.

### M5A — 실행 수직 슬라이스

- accumulator 기반 고정 물리 step과 렌더 frame 분리
- 코어 소유 runtime과 UI용 snapshot/summary 경계
- 기본 프리셋 로더, 재생·일시정지·한 스텝·리셋
- Canvas 2D 카메라, AU 축, 천체 마커, 궤적 링버퍼
- 상태바와 최소 Conservation Monitor

### M5B — 데스크톱 분석 프레임

- `tokens.css`와 TitleBar·Ribbon·DockZone·Splitter·Console·StatusBar
- SpecTree·PropertyManager·FigureDock·ConservationMonitor
- NumberField, 직교/케플러 입력 전환, 범위·유효성 검증
- 팬·줌·맞춤·선택·측정과 트리/속성/그래픽 동기화
- 에너지·각운동량·COM·분리거리 플롯
- 회전 좌표계·유효 퍼텐셜·L1–L5 오버레이

### M5C — 저장·내보내기·반응형

- 버전된 `Scenario` 스키마와 검증된 URL hash 압축/복원
- 계측 CSV 내보내기와 WebM 캡처
- 1024px 전환 레이아웃과 768px 미만 관람 모드
- 키보드 대체 조작, 포커스, 명칭, 색 이외 상태 표식
- 대표 사용자 흐름의 브라우저 E2E 테스트

### 완료 게이트

1. 프리셋 로드 → 재생/step → 선택 → 편집 → reset → 저장 → 새 세션 복원 흐름이 자동 E2E와 수동 체크리스트를 통과한다.
2. 그래픽, figure, 상태바가 모두 같은 완료된 core step의 snapshot을 표시한다.
3. URL 왕복 후 초기조건·설정이 같고 같은 CPU 경로 궤적이 결정론 허용오차 안에서 재현된다.
4. Conservation Monitor 수치가 코어 진단값과 일치하며 constrained world를 보존 성공으로 오표시하지 않는다.
5. 컴포넌트의 하드코딩 Hex가 0건이고 디자인 백서 금지 목록을 위반하지 않는다.
6. 데스크톱 기본 폭에서 가로 스크롤이 없고, 모바일 관람 모드에서 핵심 재생·보존 상태를 확인할 수 있다.
7. 키보드 포커스, 아이콘 버튼 명칭, 색 이외 오류 표식, 텍스트 대비 게이트를 통과한다.
8. 유효하지 않거나 과도하게 큰 URL payload는 로더 주입 전에 안전하게 거부된다.

### 출시 표식

이 게이트 통과 시 **Product Beta**로 표기한다.

## 10. M6 — 자연어 시나리오 연동 (선택)

### 진입 조건

M5 게이트 통과, 외부 공급자·모델·운영비·배포 방식에 대한 사용자 결정.

### 목표

자연어를 검증된 `Scenario` 후보로 변환하되 외부 서비스 실패가 코어 제품에 전파되지 않게 한다.

### 작업 패키지

- **M6-01 계약**: 공급자 독립 `ScenarioDraft` JSON Schema와 오류 형식
- **M6-02 프록시**: 별도 최소 서버 패키지, timeout·크기 제한·키 격리
- **M6-03 공급자 어댑터**: 모델별 요청/구조화 출력 파싱을 한 모듈에 격리
- **M6-04 검증·정규화**: 필수 필드, 유한값, 질량·궤도 범위, ID 충돌, 단위 명시
- **M6-05 UX**: 콘솔 요청·검토·적용 단계, 실패 사유, 프리셋/수동 경로 유지

### 완료 게이트

1. 대표 평문 입력이 schema-valid 초안으로 변환되고 사용자 적용 후 정상 적분된다.
2. 음수 질량, NaN/Inf, 누락 필드, 과대 payload, 알 수 없는 버전이 코어 주입 전에 차단된다.
3. 클라이언트 번들·로그·URL·Git 추적 파일에 API 키가 0건이다.
4. 공급자 timeout·5xx·잘못된 응답에서 프리셋·수동 편집·시뮬레이션이 정상이다.
5. 공급자 응답 형식 변경은 어댑터와 계약 fixture 이외의 코어 파일 수정을 요구하지 않는다.
6. 프록시 없이 생성한 정적 빌드가 완전히 동작한다.

### 비차단 규칙

M6 미착수·미완료·비활성은 M7과 Core v1 출시를 막지 않는다.

## 11. M7 — 성능·호환성·출시

### 진입 조건

M5 게이트 통과. M6은 선택.

### M7A — 측정과 런타임 격리

- 기준 장치·브라우저·전원 모드·뷰포트 기록
- N=100·500·2000 이상에서 physics step, render, diagnostics, trail 비용 분리 측정
- long-task, 메모리, 궤적 버퍼 상한 계측
- 필요 시 Web Worker adapter와 snapshot 전달 주기 도입

### M7B — 가속 경로 (측정 시에만)

- 직접합산 경로를 정확성 oracle로 유지
- 3D Barnes–Hut octree, 개방각 `θ`, leaf 정책, 결정적 순회 구현
- canonical distribution별 천체 가속도 diff와 단기 궤적 parity 리포트
- Barnes–Hut으로 목표를 못 맞출 때만 GPU spike를 별도 결정

가속 경로를 도입하면 `θ=0.5` 기준 p95 normalized acceleration error `≤ 1%`를 초기 목표로 두고, 기준 궤도의 보존 오차 포락선이 직접합산의 2배를 넘지 않게 한다. 분모가 거의 0인 천체는 characteristic acceleration로 정규화한다. 최종 기준은 benchmark fixture와 함께 고정한다.

### M7C — CI·호환성·배포

- 모든 변경: lint·typecheck·빠른 테스트·build
- 예약/릴리스: M3 장기 보존·M4 CR3BP·M7 parity/benchmark
- Chrome·Edge·Firefox·Safari 최신 계열의 핵심 흐름 smoke
- Canvas/WebGL 미지원 또는 가속 비활성 환경의 CPU 폴백
- 정적 배포 아티팩트, 캐시 정책, 오류 없는 직접 URL 진입

### 완료 게이트

1. 기준 장치에서 수백 천체의 60fps 목표를 측정하고 결과·설정·미달 사유를 리포트한다.
2. 가속을 도입했다면 CPU parity, 보존, 결정된 성능 기준을 모두 통과한다. 도입하지 않았다면 프로파일이 불필요함을 보여준다.
3. 고정 길이 trail/diagnostics 버퍼로 장시간 실행 시 메모리가 무한 증가하지 않는다.
4. LCP가 기술 백서 기준 `≤ 2.5s`이고 초기 번들 분석 결과가 기록된다.
5. 빠른 CI와 릴리스 검증이 모두 green이며, 장기 검증 결과를 추적할 수 있다.
6. 지원 브라우저의 핵심 E2E smoke와 CPU 폴백이 통과한다.
7. 배포 프리뷰에서 기본 프리셋, URL 복원, 재생, 계측, CSV/WebM 지원 경로가 확인된다.
8. 알려진 제한, 기준 하드웨어, 물리 허용오차, 선택 기능 상태가 릴리스 문서에 명시된다.

### 출시 표식

이 게이트 통과 시 **Core v1**로 표기한다.
